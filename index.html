Approach

Simple Data Science?

My idea is to make data science applicable and easy to understand. It doesn’t have to be rocket science. Nowadays, there are great libraries available that let us apply machine learning on many problems quick and easy, thus creating some real value for any company - and that’s what I like to do. Simple data science. No overengineering.

What About That Hype?

And let’s be honest: In some cases, Machine Learning or AI is not (yet) the right solution for a problem. Maybe a simplistic BI-dashboard that makes data from Access or Excel more transparent for everyone is the right way to go. That’s totally fine - and part of my work approach: Let’s ignore all that buzz and keep our feet on the ground.

Deploying Data Science

Data science is fine - but how do you actually make it usable for the business user?

This small project is to give an example of a modern data science product using methods such as modularization, data gathering from external sources, continuous re-training of a machine learning model, publishing and visualizing via an API - all hosted on AWS.

The main idea is to build everything around an API, and use that API for (A) controlling/triggering main fuctions such as unpdating the database, re-training the machine learning model and… (B) …making data and results available through jsons. As a result, any theoretical application accross all devices can access this data science product - as long it is connected to the web!

Exemplary application: A simple dashboard that shows live water levels as well as most recent predictions

http://ec2-18-197-132-108.eu-central-1.compute.amazonaws.com:8080/show

https://github.com/mheerens/product-api

Classical Machine Learning

This project is two things at the same time: A workflow structure for doing machine learning (in this case: building and evaluating a model), and at the same time a collection of frequently used functions. The Goal of this project is to provide a solid template that can - after customization - be used for many of the “classical classification” and/or regression problems that might occur in a hand-on business context. No fancy rocket science, but solid machine learning with a practial approach.

It is continously developed - so if I come across new and exiting things that fit in, I will definitely add them.

https://github.com/mheerens/ml_essentials
